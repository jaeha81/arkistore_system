# Plan: 나인탁 발주서 엑셀 업로드 기능

## 구현 접근 방식

DB 연결 없이 프론트엔드에서 **클라이언트 사이드 엑셀 파싱** → 프리뷰 → 기존 API로 생성하는 방식.
(DB가 아직 미설정 상태이므로 백엔드 변경 최소화)

## 변경/추가 파일

### 프론트엔드 (arki-web)

| 파일 | 변경 | 설명 |
|------|------|------|
| `package.json` | 수정 | `xlsx` 라이브러리 추가 |
| `lib/api.ts` | 수정 | `purchaseOrdersApi.uploadExcel` 추가 |
| `app/arki/purchase-orders/page.tsx` | 수정 | "엑셀 업로드" 버튼 추가 |
| `app/arki/purchase-orders/upload/page.tsx` | **신규** | 업로드 페이지 |
| `lib/parseExcel.ts` | **신규** | 엑셀 파싱 유틸 |

## 구현 흐름

```
[발주서 목록] → "엑셀 업로드" 버튼 클릭
    ↓
[업로드 페이지]
    1. 파일 선택 (드래그앤드롭 / 클릭)
    2. 클라이언트에서 xlsx 파싱
    3. 파싱 결과 프리뷰 테이블 표시
       - 상품코드, 상품명, 수량, 단가, 금액
       - 오류 행 빨간색 하이라이트
    4. "발주서 생성" 버튼 → 기존 API 호출
    5. 완료 → 발주서 상세 페이지 이동
```

## 엑셀 포맷 (기본 템플릿)

나인탁 발주서 포맷 미확인 → 범용 포맷으로 구현 후 매핑 조정 가능하게 설계

### 헤더 영역 (1~3행)
| 셀 | 내용 |
|----|------|
| A1 | 공급업체명 |
| B1 | (값) |
| A2 | 발주일 |
| B2 | (값) |
| A3 | 입고예정일 |
| B3 | (값) |

### 아이템 영역 (5행~ / 4행=헤더)
| A | B | C | D | E |
|---|---|---|---|---|
| 상품코드 | 상품명 | 수량 | 단가 | 금액 |
| PRD-001 | 소파 3인용 | 10 | 850000 | 8500000 |

### 새 파일: `upload/page.tsx`

### 새 파일: `lib/parseExcel.ts`

xlsx 라이브러리로 엑셀 파싱 + 데이터 검증 유틸

## 고려사항

1. **나인탁 발주서 포맷** — 실제 샘플 받으면 매핑 컬럼 조정 필요
2. **상품코드 매칭** — DB 없이는 프리뷰에서 코드만 표시, DB 연결 후 자동 매칭
3. **대용량 파일** — 1000행 이상 시 성능 고려 (현재 범위 밖)
4. **중복 방지** — idempotency_key 자동 생성
